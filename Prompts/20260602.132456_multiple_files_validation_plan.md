# Implementation Plan - Multiple Files Drop Validation & User Warning

This plan outlines the changes required to detect when a user attempts to drag-and-drop multiple individual files simultaneously, block the operation, and notify the user with an informational modal suggesting they package the items in a folder instead.

## User Review Required

> [!IMPORTANT]
> When multiple files are dropped:
> 1. We will block any file sharing operations completely.
> 2. We will display a clear JavaFX `Alert` modal to guide the user to collect all files into a single folder and drop that folder instead to share them together.

## Open Questions

None.

## Proposed Changes

### UI Layer

#### [MODIFY] [ShareTab.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/ui/ShareTab.java)
- Update the `setOnDragDropped` handler in `setupDragAndDrop()` to check the size of the dropped files list (`files.size()`).
- If `files.size() > 1`:
  - Show a warning Alert dialog with a friendly, descriptive message.
  - Set `success = false` and consume the event without starting any torrent creation process.
- If `files.size() == 1`:
  - Proceed with the normal `processFilesForTorrent(files.get(0))` hashing and packaging flow.

## Verification Plan

### Automated Tests
- Run `.\gradlew.bat compileJava` to ensure code compiles successfully.

### Manual Verification
- Open the "Share Files" tab.
- Select and drag **two or more individual files** simultaneously into the drop zone.
- Verify that a warning dialog pops up explaining that multiple files are not allowed and suggesting collecting them in a folder.
- Verify that no torrent metadata is created and no sharing action begins.
- Select and drag **a single folder** containing multiple files into the drop zone.
- Verify that torrent generation proceeds normally.
