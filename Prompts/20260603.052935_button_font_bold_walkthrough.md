# User Prompt
```text
the font on buttons is small and thin compared to button size.  make it big and bold
```

# Walkthrough - Make Button Font Big and Bold

## Problem
All buttons across the app had `-fx-font-size: 13px` in their CSS classes (`.button-primary`, `.button-secondary`).
Additionally, every button had an inline `setStyle("-fx-padding: 8 16;")` call in Java, which overrides CSS class properties — meaning CSS font changes were silently ignored.

## Changes Made

### [`index.css`](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/resources/index.css)
- Added a global `.button` base rule with `font-size: 15px`, `font-weight: bold`, and `font-family: Outfit/Inter`
- Updated `.button-primary` and `.button-secondary` font size from `13px` → `15px`

### [`SettingsTab.java`](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/ui/SettingsTab.java)
- Removed `setStyle("-fx-padding: 8 16;")` from `addBtn`, `deleteBtn`, `saveBtn`, `browse`

### [`AddMagnetDialog.java`](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/ui/AddMagnetDialog.java)
- Removed `setStyle("-fx-padding: 8 16;")` from `browseBtn`

### [`DownloadsTab.java`](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/ui/DownloadsTab.java)
- Removed `setStyle("-fx-padding: 8 16;")` from `addMagnetBtn`

## Root Cause
JavaFX `setStyle()` sets inline styles with highest CSS priority. Even though the CSS class had bold fonts, the inline `setStyle()` was fully overriding the class rules. Removing the inline calls lets the CSS classes control all button styling.
