# Walkthrough - Fix Share Tab Drag & Drop Events & Add System Default Styling

We have successfully resolved the file drag-and-drop issue on the "Share Files" tab and added elegant system-compliant visual styling for the drop zone.

## Changes Made

### UI Components

#### [ShareTab.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/ui/ShareTab.java)
- Configured the drag-and-drop `StackPane` with `dragZone.setPickOnBounds(true);`. This ensures that even when the background is empty or transparent, JavaFX captures the drag and mouse events correctly over the entire region instead of only directly over the text.
- Styled `dragZone` with an elegant, light-mode system-compliant Modena styling:
  - Dashed gray border using standard UI token `-fx-box-border`.
  - Background color using default standard UI token `-fx-control-inner-background`.
  - Rounded corners (`-fx-border-radius: 16px; -fx-background-radius: 16px;`) and pointer cursor.
- Updated `setupDragAndDrop()` to dynamically swap `dragZone` inline style strings:
  - When a file is dragged over, it transitions to `-fx-accent` border color and derives a slightly darker background shade using `derive(-fx-control-inner-background, -5%)`.
  - When the drag exits or is dropped, it seamlessly transitions back to standard system values.
