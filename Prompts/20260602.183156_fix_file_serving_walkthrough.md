# Walkthrough - Fix File Sharing / Seeding

## Root Cause

In [`TorrentManager.java`](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/torrent/TorrentManager.java), the `startSeeding()` method was calling:

```java
sessionManager.getJlibtorrentSession().download(ti, sourcePath.getParentFile());
```

The `download()` API tells libtorrent to **fetch data from remote peers** — the exact opposite of seeding. libtorrent had no way to know the files already existed locally, so it would try to download them and fail (or do nothing useful for serving).

## Fix Applied

Replaced `download()` with `AddTorrentParams` + `TorrentFlags.SEED_MODE`:

```java
AddTorrentParams atp = new AddTorrentParams();
atp.torrentInfo(ti);
atp.savePath(savePath.getAbsolutePath());
atp.flags(TorrentFlags.SEED_MODE);

error_code ec = new error_code();
session.swig().add_torrent(atp.swig(), ec);
```

`SEED_MODE` tells libtorrent:
- The data files already exist at `savePath`
- Skip piece re-hashing/verification
- Start announcing to trackers and accepting peer connections immediately

If libtorrent detects a hash mismatch at runtime, it automatically exits seed mode and performs a full re-check — so this is also safe.
