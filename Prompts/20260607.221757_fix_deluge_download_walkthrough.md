# Walkthrough of Changes

To allow Deluge (and other popular clients) to download from TorrentShare, we implemented three key updates to enable loopback / local IP connectivity and correct peer discovery.

### Changes Made

#### 1. libtorrent Session Configuration Update
In [SessionManager.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/torrent/SessionManager.java):
- Set `allow_multiple_connections_per_ip` to `true` to permit multiple connections from the same IP, enabling same-machine and NAT loopback transfers.
- Bound `listen_interfaces` to `"0.0.0.0:" + listeningPort + ",[::]:" + listeningPort` to listen on all IPv4 and IPv6 interfaces.

#### 2. Saved Magnet Link Retrieval Helper
Added a getter method to retrieve the fully generated and persisted magnet link from the local database:
- [TorrentManager.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/torrent/TorrentManager.java): Added `getMagnetUri(infoHash)`.
- [ShareService.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/service/ShareService.java): Added `getMagnetUri(infoHash)`.

#### 3. UI Magnet Link Synchronization
In [ShareTab.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/ui/ShareTab.java):
- Updated `populateTorrentMetadata` to query the saved magnet URI via `shareService.getMagnetUri(infoHash)`. This exposes the magnet containing direct peer address hints (`x.pe=`) to the UI, enabling instant local discovery in receiving clients.
