# Walkthrough - Multiple Files Drop Validation & User Warning

We have successfully implemented a check that detects when a user attempts to drop multiple individual files simultaneously, blocks the sharing action immediately, and presents an informational warning dialog recommending the user collect the items into a single folder and drop that folder instead.

## Changes Made

### UI Components

#### [ShareTab.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/ui/ShareTab.java)
- Updated `setOnDragDropped()` inside `setupDragAndDrop()` to inspect the size of the retrieved files list (`files.size()`).
- If `files.size() > 1`:
  - Blocks any subsequent processing and sets `success = false;`.
  - Triggers an OS-native JavaFX `Alert` modal of type `WARNING` to inform the user.
  - Informative content: *"Sharing multiple individual files directly is not supported. To share these files, please collect them into a single folder, and then drop that folder here to package and share them together."*
- If `files.size() == 1`:
  - Proceeds normally to call `processFilesForTorrent(files.get(0))` to hash and share the file or recursive folder structure.
- Declared a helper method `showWarning(String title, String details)` that spawns standard `AlertType.WARNING` alerts beautifully.

## Verification Results

### Validation Accuracy
- Dropping two or more files successfully aborts the drag action, consumes the drop event as a failure, and correctly launches the alert box.
- Dropping a single folder or a single file is fully allowed and processes seamlessly.
