# Implementation Plan - System Theme Support (Light/Dark Mode Auto-Detection)

## Goal
Detect the Windows system theme (light/dark mode) dynamically at startup, and apply a premium light or dark color scheme accordingly.

## Proposed Changes

### Helper class for theme detection
#### [NEW] [ThemeDetector.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/util/ThemeDetector.java)
- Create a helper utility to read the Windows Registry key `HKCU\Software\Microsoft\Windows\CurrentVersion\Themes\Personalize\AppsUseLightTheme`.
- If the value is `0`, system is in Dark Theme. If `1` (or if it fails/non-Windows), fallback to Light Theme.

### Inject theme class on startup
#### [MODIFY] [MainWindow.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/ui/MainWindow.java)
- Apply the style class `dark-theme` to the scene's root node if `ThemeDetector.isDarkMode()` returns true.

### Adapt stylesheet to support both themes
#### [MODIFY] [index.css](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/resources/index.css)
- Define a beautiful set of variables for light theme (default `.root`) and dark theme (`.root.dark-theme`).
- Use these variables for background, text, borders, tables, list views, and dialogs.
