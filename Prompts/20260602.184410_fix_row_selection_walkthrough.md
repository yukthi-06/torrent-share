```text
on "Active Downloads" tab, when a row is selected , it is getting unselected immediately. fix it
```

# Walkthrough - Fix Row Selection Being Cleared on Refresh

## Root Cause

`StatusRefreshTask` polls every **1 second** and calls `DownloadsTab.updateActiveDownloads()`, which calls:

```java
tableView.getItems().setAll(statuses);
```

`setAll()` **replaces the entire ObservableList**, which JavaFX treats as a full reset of the TableView — clearing the selection model every time, 1×/second.

## Fix Applied in `DownloadsTab.updateActiveDownloads()`

Instead of replacing the whole list:

1. **Save** the currently selected item's `infoHash` before update.
2. **Smart update** the items list:
   - Update existing rows in-place (same infoHash → update that slot).
   - Remove rows whose infoHash is no longer present.
   - Append new rows.
3. **Restore** the selection by finding the item with the saved `infoHash`.

This avoids the full list replacement that clears selection, while still keeping the table data current.
