# User Prompt
You are a senior Java desktop architect and principal engineer.

Build a complete production-quality desktop application called **TorrentShare**.

The application is a native desktop BitTorrent-based file sharing tool that allows users to share files and folders by generating magnet links and allows other users running the same application to download those files by pasting the magnet link.

The application must support resumable downloads, automatic seeding, DHT peer discovery, tracker support, and persistent session recovery.

Generate a complete multi-module project with all source code, build files, package structure, classes, interfaces, and implementations.

Do not generate pseudocode.

Generate compilable Java code.

# Product Goal

The application should work as follows:

## Sender Workflow

1. User launches TorrentShare.
2. User drags and drops a file or folder into the application.
3. The application creates a torrent in the background.
4. The application saves a .torrent file locally.
5. The application immediately begins seeding.
6. The application generates a complete magnet URI.
7. The application displays:

   * Magnet URI
   * Info Hash
   * File Name
   * File Size
   * Tracker List
8. User copies the magnet URI and sends it to a friend.

---

## Receiver Workflow

1. User opens TorrentShare.
2. User clicks "Add Magnet".
3. User pastes a magnet URI.
4. User selects a download directory.
5. Download starts.
6. Download progress is displayed.
7. User can pause/resume.
8. If application closes:

   * Download state is preserved.
   * Resume data is saved.
9. When application reopens:

   * All torrents are automatically restored.
   * Downloads continue from the exact point where they stopped.

No user action should be required to restore downloads.

## 📈 Resilient Fastresume and Session Recovery (The Core Goal)

To ensure that downloads survive unexpectedly terminated sessions (reboots, power outages, app crashes):
1. **Dynamic Real-Time Saves**: The app subscribes to libtorrent alert loops. When a peer sends new data, libtorrent updates the pieces index. Jlibtorrent immediately posts a `SaveResumeDataAlert`. The app catches this alert in the background, bencodes the fastresume bytes, and overwrites the local `data/resume/<infoHash>.fastresume` immediately!
2. **Database Tracking**: Every added torrent is stored in SQLite table `torrents`, capturing the active file path, added date, and original magnet link.
3. **Shutdown Flush Hook**: On close, the application triggers `prepareShutdown()`. It iterates active downloads, requests flush disk cache flags from handles, waits up to 3 seconds for active write flushes to finish, and shuts down the native session cleanly.
4. **Startup Auto-Restore**: On boot, the application reads SQLite records. For each record, if `.fastresume` exists on disk, it initializes an `AddTorrentParams` directly from the bencoded bytes and loads it into the session. The download resumes at the exact sector offset without re-hashing or double-downloading! If the `.fastresume` file was corrupted, it falls back to parsing backup `.torrent` files or magnet links, guaranteeing 100% data durability.

## 🧪 Manual Verification Instructions

To launch and run the application locally on Windows:
1. Make sure you have **Java 21** installed (`java --version`).
2. Run compilation using your IDE (e.g. IntelliJ or Eclipse) or compile by running:
   ```cmd
   gradle compileJava
   ```
3. Run the development instance of the application using:
   ```cmd
   gradle run
   ```
4. **Test Sender Flow**: Drag a large video/archive file into the **Share Files** drop zone. The progress panel will display hashing progress in the background. Once finished, click "Display QR Code" to inspect the Matrix PNG or copy the magnet URI! The application is now actively seeding the file locally.
5. **Test Receiver Flow**: Open a second instance of the app (or test on the same client). Click "+ Add Magnet Link", paste your copied link, and choose a save directory. The client will fetch the torrent metadata from your seeding task, and start downloading files.
6. **Test Resiliency**: Close the application during the download. Check that a fastresume file was generated under `data/resume/*.fastresume`. Restart the app—the downloads tab will automatically restore your torrent tasks and resume progress.
