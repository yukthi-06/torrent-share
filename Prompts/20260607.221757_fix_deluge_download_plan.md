# User Prompt
seeding from this app, i am not able to download from popular torrent client Deluge

# Goals
Enable successful connection and download from popular BitTorrent clients like Deluge when seeding from this application. This is particularly important for local/LAN transfers or testing on the same machine/NAT.

## Proposed Changes

### jlibtorrent Session Configuration

#### [MODIFY] [SessionManager.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/torrent/SessionManager.java)
- Enable `allow_multiple_connections_per_ip` to allow multiple connections from the same IP (vital for same-machine and behind-NAT/VPN loopback testing).
- Update `listen_interfaces` to bind to both IPv4 and IPv6 wildcards (e.g., `0.0.0.0:port,[::]:port`) to support modern dual-stack client connections.

### Share UI Magnet URI Synchronization

#### [MODIFY] [TorrentManager.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/torrent/TorrentManager.java)
- Expose a helper method `getMagnetUri(String infoHash)` to retrieve the persisted magnet URI (which contains correct custom trackers and local peer address hint `x.pe`) from the database.

#### [MODIFY] [ShareService.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/service/ShareService.java)
- Delegate `getMagnetUri(String infoHash)` to `TorrentManager`.

#### [MODIFY] [ShareTab.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/ui/ShareTab.java)
- Instead of generating a generic magnet link without trackers or peer hints in `populateTorrentMetadata`, retrieve the fully populated magnet URI via `shareService.getMagnetUri(infoHash)`.

## Verification Plan

### Automated Tests
- Run Gradle clean and compile to verify Java compilation:
  `cmd /c gradlew.bat compileJava`

### Manual Verification
- Share a file or folder in TorrentShare.
- Copy the generated magnet URI containing the local network peer hint.
- Paste the magnet link into Deluge (on the same machine or a local network peer).
- Verify that Deluge successfully connects to the TorrentShare peer and downloads the files.
