# Implementation Plan - Fix Tracker Settings Panel Only Showing 4 Values

## Root Cause Analysis

The `loadDefaultTrackers()` method in `MagnetUtils.java` uses `java.util.Properties.load(InputStream)` to parse `trackers.properties`. This method:
1. Interprets `\` as an escape character — which is also used as the multi-line continuation marker in the file.
2. Silently swallows any exception in the catch block, falling back to the **hardcoded 4-tracker list**.

If `Properties.load()` encounters any parsing issue (e.g., the `:` in `udp://` being interpreted as a key-value separator in some edge case, or the escape handling behaving unexpectedly), the exception is silently caught and the 4-item fallback is returned — which is exactly what the user sees in the settings panel.

## Fix

Replace `Properties.load()` with a **manual line-by-line reader** in `MagnetUtils.loadDefaultTrackers()` that:
- Skips comment lines (starting with `#`).
- Strips the `trackers=` prefix from the first content line.
- Handles multi-line continuation (`\` at end of line) by joining lines.
- Splits the joined value by `,` and trims each entry.
- Logs the count of loaded trackers for observability.
- Logs the actual exception on failure instead of silently swallowing it.

## Files Changed

### [MODIFY] MagnetUtils.java
- Rewrite `loadDefaultTrackers()` to use `BufferedReader` + manual parsing instead of `Properties.load()`.
- Add `Logger` for error reporting so failures are visible in logs instead of silent.
