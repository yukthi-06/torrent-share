# Walkthrough - Fix Metadata Discovery Behind CG-NAT

## Problem
The previous fix embedded the sender's LAN IP as `x.pe=` in the magnet link. Behind CG-NAT, the private LAN IP (e.g. `192.168.x.x`) is unreachable from the internet. For internet users, peer discovery must rely on DHT and public trackers.

The real issue causing "Fetching Metadata..." to persist was the **tracker announce delay**: when a seeder starts, libtorrent schedules the first tracker announcement at the tracker-defined interval (can be 30+ minutes). Until that announce fires, no tracker knows the seeder exists, and the receiver can't find it.
