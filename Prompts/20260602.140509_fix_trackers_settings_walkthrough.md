# Walkthrough - Fix Trackers Settings Panel (Plain Text Format)

## What Was Changed

### 1. `trackers.properties` — Converted to plain text
- Removed the `trackers=\` key-value header and all backslash line continuations.
- Now just a plain text file: **one tracker URL per line**, `#` for comments.
- All 14 trackers are listed, one per line.

### 2. `MagnetUtils.java` — Simplified `loadDefaultTrackers()`
- Replaced the complex `Properties.load()` / manual multi-line parser with a dead-simple `BufferedReader` loop.
- Logic: skip blank lines and lines starting with `#`, add everything else as a tracker URL.
- Added a `Logger` so failures are visible in logs instead of being silently swallowed.

## Root Cause of the Bug

`java.util.Properties` treats `:` as a key-value separator and `\` as an escape character. The old `trackers.properties` used `\` for multi-line continuation, but `Properties.load()` was silently throwing an exception (due to the URLs containing `://`), which was caught and swallowed, falling back to the hardcoded 4-tracker list.

## Result

- Settings panel will now show all **14 trackers** from `trackers.properties`.
- On next startup, `DatabaseManager.seedDefaultTrackers()` will sync the database (delete stale default entries, insert all 14) automatically.
