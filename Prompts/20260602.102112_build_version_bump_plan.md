# TorrentShare Dependency Version Bump Plan

This document outlines the change to downgrade the `jlibtorrent` library version in our Gradle configurations to a fully resolved release version hosted in the official FrostWire Maven repository.

## Findings
The previous version parameter (`2.0.13.6`) is not fully indexed or hosted inside `https://dl.frostwire.com/maven`. When Gradle tries to fetch it, the repository server redirects the request to `http://www.frostwire.com/downloads/?from=404`. Because the redirect goes from `https` to `http`, Gradle throws a security protocol mismatch exception.

By updating the version to `2.0.12.9` (the current stable indexing in FrostWire's official Maven repository), the server will return a `200 OK` response directly over `https` without any insecure HTTP redirects.

## Proposed Changes

### Build and Configurations

#### [MODIFY] [build.gradle](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/build.gradle)
Bump `jlibtorrent` version to `2.0.12.9`.

```diff
 dependencies {
     // Core jlibtorrent bindings and platform-specific native libraries
-    implementation 'com.frostwire:jlibtorrent:2.0.13.6'
-    implementation 'com.frostwire:jlibtorrent-windows:2.0.13.6'
+    implementation 'com.frostwire:jlibtorrent:2.0.12.9'
+    implementation 'com.frostwire:jlibtorrent-windows:2.0.12.9'
```
