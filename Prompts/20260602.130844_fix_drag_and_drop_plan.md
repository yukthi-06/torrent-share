# Implementation Plan - Fix Share Tab Drag & Drop Events & Add System Default Styling

This plan outlines the changes required to resolve the broken file drag-and-drop functionality on the "Share Files" tab and restore an elegant system-default visual boundary for the drop area.

## User Review Required

> [!IMPORTANT]
> The drag-and-drop functionality stopped capturing events on empty areas because disabling the dark stylesheet made the background of the drag zone `StackPane` completely null/transparent. In JavaFX, a transparent background on a pane causes mouse and drag-over events to pass through to parent elements.
>
> We will resolve this by:
> 1. Setting `dragZone.setPickOnBounds(true);` to ensure the entire bounds of the drag area are sensitive to drag actions.
> 2. Setting system-compliant Modena styling (light background, dashed gray border) so that the drop area is visually distinct.
