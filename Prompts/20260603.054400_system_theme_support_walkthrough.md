# Walkthrough - System Theme Support (Light/Dark Mode Auto-Detection)

## Goal
Detect and respect the operating system's light/dark mode preference on startup, rather than hardcoding a dark theme.

## Changes Made

### [`ThemeDetector.java`](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/util/ThemeDetector.java) [NEW]
- Added a utility class to detect the Windows system theme by querying the registry:
  `HKCU\Software\Microsoft\Windows\CurrentVersion\Themes\Personalize\AppsUseLightTheme`.
- If the value is `0`, dark mode is active. Otherwise, it defaults to false (light mode).

### [`MainWindow.java`](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/ui/MainWindow.java)
- Imported `ThemeDetector`.
- Conditionally added the `dark-theme` style class to the root `BorderPane` layout if the OS theme is detected as dark mode:
  ```java
  if (ThemeDetector.isDarkMode()) {
      root.getStyleClass().add("dark-theme");
  }
  ```

### [`index.css`](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/resources/index.css)
- Restructured CSS to define variables for both light and dark modes:
  - Default `.root` sets light mode color variables (backgrounds, text, borders, lists).
  - `.dark-theme` selector overrides those color variables for dark mode.
- Styled components (buttons, labels, text-fields, table-views, list-views, scrollbars) using CSS variables so they transition cleanly depending on the active system theme.
- Maintained modern layout enhancements (Outfit font family, bold 15px buttons, and spacing).
