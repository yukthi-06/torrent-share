# Walkthrough - Fix "Stuck at Metadata" When Receiving Files

## Root Cause

In `TorrentManager.magnetWithAllTrackers()`, the original code was:

```java
List<String> activeTrackers = trackerManager.getTrackers();
return MagnetUtils.generateMagnet(hash, "", activeTrackers);
```

This **completely discards** all `tr=` tracker parameters from the sender's magnet link and replaces them with only the local database trackers. The sender's specific tracker announcements — which are the most likely way to find the seeder — were being silently thrown away.

The receiver then only attempts to reach the 14 public trackers (opentrackr, openbittorrent, etc.), none of which have ever seen this brand-new torrent. Without the sender's tracker reference, libtorrent can only try DHT (slow) or LSD (LAN-only), causing the torrent to stay stuck at "Fetching Metadata..." for a very long time or indefinitely.

Additionally, the `dn=` display name from the original magnet was also discarded (replaced with `""`).

## Changes Made

### [`MagnetUtils.java`](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/util/MagnetUtils.java)
Added two new helper methods:
- `extractDisplayName(String magnet)` — parses and URL-decodes the `dn=` parameter
- `extractTrackers(String magnet)` — parses all `tr=` parameters into a list

### [`TorrentManager.java`](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/torrent/TorrentManager.java)
Fixed `magnetWithAllTrackers()`:
- Extracts and preserves the `dn=` display name from the original magnet
- Merges: **original trackers first** (sender's announcements), then local DB trackers, using `LinkedHashSet` for deduplication
- The sender's trackers take priority in the list — they are most likely to have the metadata

### [`TorrentMonitor.java`](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/torrent/TorrentMonitor.java)
- Changed `"Metadata"` → `"Fetching Metadata..."` for a clearer user-facing status label
