# Implementation Plan - Restoring High-Contrast Text Color Resolution

This plan outlines the changes required to replace the `-fx-text-background-color` inline style calls with Modena's base high-contrast color token `-fx-text-base-color`, ensuring all labels inside light containers are fully visible and readable.

## User Review Required

> [!IMPORTANT]
> The labels became invisible because JavaFX's `-fx-text-background-color` variable is resolved dynamically based on a node's *own* background fill. When applied inline to `Label` elements (which have no explicit background fills themselves and are nested inside transparent parents), JavaFX fails to compute the contrast, causing the text to render as completely transparent/invisible.
>
> We will resolve this by explicitly setting the text fill to **`-fx-text-base-color`**, which is JavaFX's globally resolved high-contrast base text color (automatically resolving to high-contrast dark-gray/black in light/Modena setups).
