# Implementation Plan - Externalize Tracker List to Properties File

This plan outlines the changes required to move all hardcoded Torrent announce tracker URLs out of the Java source code and into an external properties resource file `trackers.properties`. We will also expand the default tracker list to enhance peer discovery.

## User Review Required

> [!NOTE]
> All default announce trackers will be loaded dynamically from `trackers.properties` on class initialization.
> A robust fallback mechanism will be preserved in the Java source so that in the rare event of a resource load error, the app continues to operate normally with baseline trackers.

## Open Questions

None.

## Proposed Changes

### Configuration Layer

#### [NEW] [trackers.properties](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/resources/trackers.properties)
- Contains a comma-separated list of highly reliable, public announce trackers. (Already created in the previous step).

---

### Utility Layer

#### [MODIFY] [MagnetUtils.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/util/MagnetUtils.java)
- Add a new static method `loadDefaultTrackers()`:
  - Dynamically loads `trackers.properties` via `getResourceAsStream()`.
  - Splits the comma-separated tracker URLs and returns them as a `List<String>`.
  - Provides a baseline fallback list in case of standard I/O loading exceptions.

---

### UI Layer

#### [MODIFY] [ShareTab.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/ui/ShareTab.java)
- Update `populateTorrentMetadata()` to call `MagnetUtils.loadDefaultTrackers()` instead of hardcoding tracker URLs in the `List.of(...)` call.

---

### Persistence Layer

#### [MODIFY] [DatabaseManager.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/persistence/DatabaseManager.java)
- Update `seedDefaultTrackers()` to call `MagnetUtils.loadDefaultTrackers()` instead of hardcoding tracker URLs when initializing the default database trackers.
