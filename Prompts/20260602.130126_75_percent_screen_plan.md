# Implementation Plan - Dynamic Startup Size (75% Screen Size)

This plan outlines the changes required to start the TorrentShare application window dynamically at 75% of the primary monitor's visual dimensions.

## User Review Required

> [!NOTE]
> The app window will dynamically adapt its initial dimensions to exactly **75% of the primary monitor's visual bounds** (excluding taskbar/system dock), providing an elegant and proportional default size on any display.

## Open Questions

None.

## Proposed Changes

### UI Layer

We will dynamically compute the starting size based on the visual bounds of the primary screen.

---

#### [MODIFY] [MainWindow.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/ui/MainWindow.java)

We will modify [MainWindow.java](file:///x:/Projects_X/0_Active/1_Java_Active/Torrent_Share_Java_GITHUB/src/main/java/com/vypeensoft/torrentshare/ui/MainWindow.java) to:
- Import `javafx.stage.Screen` and `javafx.geometry.Rectangle2D`.
- Query primary screen visual bounds using `Screen.getPrimary().getVisualBounds()`.
- Calculate `defaultWidth = bounds.getWidth() * 0.75` and `defaultHeight = bounds.getHeight() * 0.75`.
- Construct the `Scene` with `defaultWidth` and `defaultHeight` instead of the hardcoded `1200` and `800`.
